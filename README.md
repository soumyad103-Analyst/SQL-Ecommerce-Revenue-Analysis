# SQL-Ecommerce-Revenue-Analysis
This project simulates a Business Analyst role in an e-commerce company. The objective was to analyze revenue performance, customer contribution, and product category trends using SQL.

# Dataset Structure
Relational database with 4 tables:
customers
orders
order_items
products
Primary and foreign key relationships were implemented to simulate real-world transactional data.

# Business Objectives
Identify top revenue-generating customers
Calculate Average Order Value (AOV)
Analyze category-wise revenue distribution
Measure impact of cancelled orders
Evaluate revenue concentration patterns

# SQL Concepts Used
Multi-table JOINs
GROUP BY & ORDER BY
Conditional filtering (WHERE)
Conditional aggregation using CASE
KPI calculation (AOV, revenue share)

# Key Insights
Revenue was concentrated among a small group of repeat customers
Electronics category contributed majority of total sales
Cancelled high-value orders created revenue leakage
AOV provided insight into purchase behavior

# Business Recommendations
Introduce loyalty program for high-value customers
Monitor cancellation patterns for premium products
Focus marketing on high-performing categories
